💰 Cálculos Financieros Fundamentales

📐 Fórmulas Básicas de Operaciones

Monto de la operación:
Monto = Cantidad × Precio unitario
Comisión en dólares:
Comisión $ = Monto × (Porcentaje de comisión / 100)
Costo total de compra:
Costo Total = Monto + Comisión
Ingreso neto de venta:
Ingreso Neto = Monto - Comisión

🔢 Ejemplo práctico de cálculo:

Compra de 0.5 BTC a $45,000

Paso 1: Calcular monto
Monto = 0.5 × $45,000 = $22,500

Paso 2: Calcular comisión (supongamos 0.5%)
Comisión = $22,500 × 0.005 = $112.50

Paso 3: Calcular costo total
Costo Total = $22,500 + $112.50 = $22,612.50

📊 Estado del Portafolio (Por cada Criptomoneda)

Tu sistema debe mantener y calcular para CADA una de las 5 criptomonedas:

📈 Variables y Cálculos por Criptomoneda

Cantidad actual en portafolio:
Cantidad Actual = Cantidad Comprada - Cantidad Vendida
Precio promedio de compra:
Precio Promedio Compra = Dinero Total Invertido / Cantidad Total Comprada
Precio promedio de venta:
Precio Promedio Venta = Dinero Total Recuperado / Cantidad Total Vendida
Ganancia/Pérdida realizada:
Ganancia = Dinero Recuperado - (Cantidad Vendida × Precio Promedio Compra)
Rentabilidad porcentual:
Rentabilidad % = ((Precio Promedio Venta - Precio Promedio Compra) / Precio Promedio Compra) × 100
⚠️ Importante: Las comisiones NO se incluyen en el dinero invertido ni en el dinero recuperado. Se llevan por separado.

Domingo 1.0% Domingo - Mínima liquidez

⏰ Ajuste Adicional por Período del Día

Período Ajuste Adicional Razón
Mañana (6-11) +0% Horario óptimo
Tarde (12-17) +0.2% Alta volatilidad
Noche (18-23, 0-5) +0.5% Baja liquidez

🧮 Fórmula de Comisión Total

Comisión Total % = Comisión Base (por día) + Ajuste Adicional (por período)

📈 Ejemplo de Venta: Calculando Ganancias

Contexto: Ya tienes 5000 ADA compradas a precio promedio de $0.262

Operación: Venta de 3000 ADA a $0.45 - Lunes a las 10 AM

PASO 1: Calcular monto y comisión
• Monto = 3000 × $0.45 = $1,350
• Período: Mañana (hora 10)
• Comisión: 0.5% (Lunes) + 0% (Mañana) = 0.5%
• Comisión $: $1,350 × 0.005 = $6.75
PASO 2: Actualizar estado del portafolio
• Cantidad vendida: 0 + 3000 = 3000 ADA
• Cantidad actual: 5000 - 3000 = 2000 ADA
• Dinero recuperado: $0 + $1,350 = $1,350
• Precio promedio venta: $1,350 / 3000 = $0.45
PASO 3: Calcular ganancia realizada
• Costo de las 3000 vendidas: 3000 × $0.262 = $786
• Ganancia = Dinero recuperado - Costo
• Ganancia = $1,350 - $786 = $564 ✅
PASO 4: Calcular rentabilidad porcentual
• Rentabilidad = ((Precio venta - Precio compra) / Precio compra) × 100
• Rentabilidad = (($0.45 - $0.262) / $0.262) × 100
• Rentabilidad = 71.76% 🎉
PASO 5: Evaluar alertas
• Rentabilidad: 71.76% (es mayor a 50%) ✓
• Alerta: "🎉 Excelente operación con ADA"

📊 Cálculo de Precio Promedio al Comprar Múltiples Veces

🔄 Cuando compras más de una cripto que ya tienes:

Nuevo Precio Promedio = Total Invertido / Total Comprado

Ejemplo práctico:

Primera compra: 2000 ADA a $0.28

  • Dinero invertido: $560
  • Precio promedio: $0.28

Segunda compra: 3000 ADA a $0.25

  • Dinero invertido adicional: $750
  • Total invertido: $560 + $750 = $1,310
  • Total comprado: 2000 + 3000 = 5000 ADA
  • Nuevo precio promedio: $1,310 / 5000 = $0.262 ← ¡Bajó!

🎯 Tareas a Desarrollar

Para implementar este sistema de gestión de portafolio de criptomonedas, debes realizar las siguientes actividades:

📝 1. Definir Atributos

Identifica todos los datos que el sistema necesita almacenar y que NO dependen de cálculos.

Recuerda: Los atributos son información base que se guarda directamente.

⚙️ 2. Definir Métodos

Identifica todas las acciones, cálculos y procesos que el sistema debe realizar.

Recuerda: Los métodos son las funciones que ejecuta la clase.

🔄 3. Definir Operaciones

Identifica los datos calculados que dependen de otros datos del sistema.

Recuerda: Las operaciones se derivan de atributos u otras operaciones.

📐 4. Diagrama de Clases

Diseña el diagrama UML que incluya:

  • Todas las clases del sistema
  • Atributos y métodos de cada clase
  • Relaciones entre clases
  • Tipos de datos

⚠️ Importante:

Antes de programar, asegúrate de:

  • Comprender completamente todas las fórmulas y cálculos
  • Identificar todas las estructuras de decisión necesarias (if, switch/case)
  • Planificar cómo almacenar y actualizar la información de cada criptomoneda
  • Diseñar la lógica para las alertas específicas por criptomoneda
  • Definir cómo se calcularán las comisiones según día y período

📋 Guía para Identificar Atributos, Operaciones y Métodos

🤔 ¿Cómo distinguir entre Atributo, Operación y Método?

📦 ATRIBUTO:
  • Son datos que NO dependen de nada
  • Se almacenan directamente sin necesidad de calcularlos
  • Representan información base del sistema
  • Pertenecen a la clase y persisten durante toda la ejecución
🔄 OPERACIÓN:
  • Son datos que dependen de otros datos
  • Se calculan o derivan a partir de atributos u otras operaciones
  • Requieren de información previa para obtener su valor
⚙️ MÉTODO:
  • Son las cosas que debe hacer la clase
  • Acciones, procesos, cálculos o funcionalidades
  • Representan el comportamiento del sistema

🔍 Diferencia CLAVE: Atributos vs Parámetros de un Método

⚠️ ¡Importante! No confundir Atributos con Parámetros:

📦 ATRIBUTOS de la Clase

  • Alcance: Toda la clase
  • Vida: Durante toda la ejecución del programa
  • Visibilidad: Accesibles desde cualquier método de la clase
  • Uso: Guardan el ESTADO del objeto
  • Dónde se definen: Al inicio de la clase

🎯 PARÁMETROS de un Método

  • Alcance: Solo dentro del método
  • Vida: Solo mientras se ejecuta el método
  • Visibilidad: Solo dentro de ese método específico
  • Uso: Reciben DATOS de entrada para el método
  • Dónde se definen: Entre paréntesis del método

💡 Ejemplo Conceptual (NO del sistema):

Clase Calculadora:

// ATRIBUTO - pertenece a la clase, persiste
resultadoAcumulado = 0

// MÉTODO con PARÁMETROS - solo existen durante la ejecución del método
sumar(numero1, numero2) {
    // numero1 y numero2 son PARÁMETROS (temporales)
    // resultadoAcumulado es ATRIBUTO (permanente)
    resultadoAcumulado = numero1 + numero2
}

📋 ¿Cómo decidir si algo es Atributo o Parámetro?

Pregunta: Respuesta:
¿Este dato debe recordarse entre diferentes operaciones? SÍ → ATRIBUTO
NO → PARÁMETRO
¿Este dato cambia solo durante una operación específica? SÍ → PARÁMETRO
NO → ATRIBUTO
¿Este dato se usa en varios métodos de la clase? SÍ → ATRIBUTO
NO → puede ser PARÁMETRO
¿Este dato se recibe desde fuera para hacer un cálculo? SÍ → PARÁMETRO
NO → ATRIBUTO

🎯 Para tu Sistema de Criptomonedas:

  • Ejemplo de ATRIBUTO: Datos acumulados que deben persistir entre operaciones (cantidades totales, montos acumulados, etc.)
  • Ejemplo de PARÁMETRO: Datos de entrada temporal para un cálculo específico (valores que se reciben para una operación puntual)
  • Piensa bien: ¿Este dato debe recordarse después de terminar una operación? Si SÍ → ATRIBUTO. Si NO → puede ser PARÁMETRO
